C1 — Dev-Tools aus GitHub entfernt: scripts, tests, release, archive internals, db-Snapshots

Der Morgen begann mit einem mulmigen Gefühl — der radikale Schnitt stand bevor. Die Dev-Tools mussten aus GitHub verschwinden, aber nicht von der Platte. Der Befehl dafür war git rm --cached. Anders als git rm ohne Flag, das die Datei sowohl aus dem Index als auch von der Festplatte entfernt, belässt --cached die Arbeitskopie intakt. Der Unterschied ist entscheidend für den Betrieb: die Scripts bleiben lokal ausführbar, aber verschwinden aus dem nächsten Commit. Damit der Effekt dauerhaft ist, wandern die betroffenen Pfade zusätzlich in die .gitignore, sodass sie auch bei zukünftigen git add-Aufrufen nicht versehentlich wieder ins Tracking rutschen. Die .gitignore selbst ist ein zeilenbasierter Filter den git bei jedem git add auswertet — jedes Muster wird gegen den relativen Pfad jeder Datei geprüft. Die Reihenfolge spielt eine Rolle: spätere Muster können frühere überschreiben, und ein ! am Zeilenanfang hebt eine Ignorierung auf. Die Muster sind hierarchisch: core/scripts/ ignoriert alles im Verzeichnis, V70/ ignoriert das ganze Versionierungs-Referenz-Verzeichnis. Ausnahme: Muster die mit ! beginnen heben die Ignorierung auf — core/archive/assets/ wurde später mit ! wieder aktiviert, weil die README-Bilder sichtbar bleiben müssen.
