## Verhältnisse-System (Attitudes) für alle 14 Narratoren

Jeder Commit-Erzähler hat jetzt individuelle Dispositionen — sieben Werte (0-10)
die bestimmen, WAS ihn an diesem Commit stört oder freut:

| Dimension | Bedeutung |
|-----------|----------|
| code_love | Freude an sauberem Code |
| cleanup_resentment | Genervtheit vom Hinterherräumen |
| doku_irritation | Wie sehr Doku-Arbeit nervt |
| criticism_tendency | Neigung zu kritisieren |
| praise_tendency | Neigung zu loben |
| verbosity_bias | Extra-Wortschwall-Tendenz |
| optimism | Glas halbvoll vs. halbleer |

Beispiele: Buffy (code_love=8, criticism=8) liebt eleganten Code und meckert
trotzdem. Spark (code_love=9, praise=9) findet ALLES super. Null (optimism=0,
praise=0) glaubt an gar nichts. Argos (cleanup_resentment=9, criticism=9) hat
es satt, anderer Leute Müll wegzuräumen.

## Mood-Overlay (attitude_modifiers in narrative_params.json)

10 Moods modifizieren die Basis-Werte mit Deltas (-4 bis +4).
Z.B. "triumphierend" boostet praise +4 und senkt criticism -3.
"erschöpft" boostet cleanup_resentment +3 und senkt optimism -3.
"alarmiert" senkt optimism um satte -4 (größter Einzel-Delta).

Endwerte werden auf 0-10 geclampt — ein "triumphierender" Null erreicht
trotzdem nur praise=4/10 (Basis 0 + 4 = 4).

## derive_composite.js

Berechnet finale Attitudes (Basis + Mood-Modifier) und gibt sie im
Konsolen-Output aus — das LLM liest sie als Prompt-Kontext.
