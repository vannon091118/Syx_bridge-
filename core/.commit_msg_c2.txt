Nach C1 der die Dev-Tools mit git rm --cached aus dem Index entfernt hatte, war die .gitignore zerrissen wie ein Schlachtfeld. Drei Versionen derselben Datei kämpften um die Vorherrschaft, resultierend in Konfliktmarkierungen die aussahen wie <<<<<<< HEAD gefolgt von der ersten Version, dann eine Reihe von ======= die sie von der zweiten Version trennte, und schliesslich >>>>>>> der Name des anderen Branches. Git verwendet diesen Drei-Wege-Merge-Algorithmus: es nimmt die gemeinsame Basis (merge base), vergleicht sie mit beiden Änderungen, und wenn beide Seiten dieselbe Zeile unterschiedlich geändert haben, produziert es einen Konflikt. Die .gitignore ist besonders anfällig dafür, weil sie zeilenbasiert ist und jede Zeile unabhängig von den anderen existiert. Das !-Präfix das eine Ignorierung aufhebt, wird von git als separate Änderung behandelt — wenn eine Seite die Zeile löscht und die andere sie modifiziert, ist ein Konflikt unvermeidbar.
